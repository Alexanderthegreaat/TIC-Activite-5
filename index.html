Les TIC (Techniques de l'Information et de la Communication)

Les TIC regroupent les outils et technologies utilisés pour manipuler, transmettre et échanger des informations. Elles combinent l'informatique, les télécommunications, l'électronique et l'audiovisuel. Les TIC permettent la communication, l'accès à l'information et la production de médias via des dispositifs informatiques, des réseaux et l'Internet. Elles ont des caractéristiques telles que la rapidité, la décentralisation du pouvoir et l'accès mondial aux informations. Les objectifs de communication des TIC sont de fournir des informations, d'influencer les perceptions et de stimuler les comportements. Les médias, tels que la presse, la radio, la télévision et Internet, sont utilisés pour diffuser des informations de manière large et collective, tandis que le multimédia permet l'utilisation simultanée de divers modes de représentation de l'information.

Réseau (informatique)

Un réseau informatique est un ensemble d'ordinateurs connectés pour échanger des informations. Il a une architecture physique et logique, un débit de transmission et une distance maximale. Les termes tels que station de travail, nœud et paquet sont utilisés dans les réseaux. Les réseaux peuvent être homogènes ou hétérogènes, et comprennent des ordinateurs, des logiciels, des supports de connexion et des équipements d'interconnexion. Ils sont classés selon leur portée, leur utilisation et leur support de connexion.

Composants matériels d'un réseau informatique

Les composants matériels d'un réseau incluent les cartes réseau (NIC) pour la connexion, les switches pour la communication entre appareils et les hubs pour la connexion multiple. Le modèle OSI divise les communications en sept couches, tandis que le modèle TCP/IP se compose de quatre couches principales. L'adressage IP identifie de manière unique les ordinateurs dans un réseau, avec IPv4 et IPv6 comme types d'adresses. Les types d'adresses IPv4 comprennent l'adresse réseau et l'adresse de diffusion.

Introduction au Web et à Internet

Le web et internet sont pareils. Le WWW (World Wide Web) est un système hypertexte public sur Internet. Le Web permet de consulter des pages accessibles sur des sites en utilisant un navigateur. Le Web est une application d'Internet, inventée par Tim Berners-Lee et Robert Cailliau.

1) Aspects techniques du Web:

Les trois mécanismes de base du Web sont :

  1. URL : Lien hypertexte permettant d'accéder à une page en spécifiant son adresse sur le serveur.
  2. HTTP : Protocole de communication utilisé pour le transfert des données entre le client et le serveur.
  3. HTML : Langage de description de pages Web, utilisé pour structurer et afficher le contenu des pages.

2) Page web et site web:

Une page web est un document multimédia contenant du texte, des images, des liens et des possibilités d'interaction. Elle est décrite à l'aide du langage de description HTML et est affichée par un navigateur. Un site web est un regroupement de pages web autour d'une thématique commune, reliées entre elles par des liens hypertextes. Il est structuré sous forme d'arborescence de dossiers et de fichiers.

3) Création d'une page web HTML:

HTML (HyperText Markup Language) est un langage de description de pages web. Un document HTML est un fichier texte structuré par des balises. Il est interprété par un navigateur et peut être créé dans un éditeur de pages web ou un éditeur de texte. Les balises HTML délimitent différentes parties du document, telles que l'en-tête, le titre et le corps. Les titres et les paragraphes sont définis par des balises spécifiques.

Exemple d'un fichier HTML :


<!DOCTYPE html>
<html>
<head>
  <title>Exemple</title>
</head>
<body>
  <h1>Hello world</h1>
</body>
</html>

Git, GitHub et la gestion de version

Dans ce chapitre, nous allons définir Git et GitHub, expliquer pourquoi Git est populaire et pourquoi les systèmes de gestion de version sont essentiels aujourd'hui.

  • Qu'est-ce que Git?

    • Git est un logiciel de versioning créé en 2005 par Linus Torvalds, le créateur de Linux. Il permet de conserver un historique des modifications apportées à un projet, facilitant ainsi l'identification des changements et la restauration des versions antérieures en cas de problème. Git est largement utilisé dans la gestion de projets et permet de travailler efficacement en équipe.

  • L'importance d'un système de gestion de version

    • Imaginons que vous possédiez un site web et que vous souhaitiez apporter des modifications ou tester de nouvelles fonctionnalités. Sans un système de gestion de version, vous devriez effectuer des sauvegardes manuelles du site avant chaque modification, ce qui deviendrait fastidieux. De plus, si plusieurs personnes travaillent sur le même projet, il est essentiel de coordonner les efforts, d'éviter les conflits et de disposer de versions à jour du projet pour chaque développeur.

  • Les deux modèles de gestion de version: centralisé vs décentralisé

    • Les systèmes de gestion de version peuvent suivre deux modèles: centralisé ou décentralisé. Le modèle centralisé implique un serveur distant contenant le code source du projet, tandis que les utilisateurs se connectent à ce serveur pour travailler dessus. En revanche, le modèle décentralisé, popularisé par Git, permet à chaque utilisateur de télécharger et d'héberger l'intégralité du code source du projet sur sa propre machine. Le modèle décentralisé offre plus de flexibilité et de sécurité.

  • Qu'est-ce que GitHub?

    • Git est un logiciel de gestion de version, tandis que GitHub est un service d'hébergement en ligne de dépôts Git. GitHub est le plus grand hébergeur de dépôts Git au monde, et de nombreux dépôts hébergés sur GitHub sont publics, permettant à d'autres contributeurs de télécharger le code et de proposer des modifications.

  • Utilisation de Git: ligne de commande vs interface graphique

    • Git peut être utilisé via une interface en ligne de commande ou une interface graphique. Ce cours se concentrera sur l'utilisation de la ligne de commande, car elle offre un accès complet à toutes les fonctionnalités de Git et garantit une expérience cohérente pour tous les utilisateurs. La connaissance de la ligne de commande facilite également l'utilisation d'autres interfaces graphiques.

  • Installation de Git

    • La manière la plus simple d'installer Git est de télécharger la dernière version à partir du site officiel
    "http://git-scm.com/downloads" et de suivre les instructions d'installation par défaut.

  • Configuration de Git

    • Après l'installation de Git, il est recommandé de configurer certaines informations, telles que le nom d'utilisateur et l'adresse e-mail, afin de les utiliser par défaut dans tous les projets Git. Cela peut être fait en utilisant la commande "git config --global user.name" suivie de votre nom et "git config --global user.email" suivi de votre adresse e-mail.

  • Démarrage d'un dépôt Git

    • Un dépôt Git est la copie d'un projet dans Git. Il peut être créé en important un répertoire local existant ou en clonant un dépôt distant
